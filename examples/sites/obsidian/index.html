Obsidian vault → Alap menus

This isn't a doc reader — docs.alap.info has that, with proper VitePress search. This page is a demo of the :obsidian:core: protocol: the same expression machinery works against any vault you point it at — your own notes, a team wiki, a shared project folder. The vault here is a mix of the project's own docs and a few small topical sources (birds, cooking, films, novels, Bach, and an untrusted-content sample), all real markdown already on disk. Clone the repo, point vaultPath at your own vault, and every trigger on this page resolves against your content instead.

Core vs REST — which mode and why

Two ways to wire :obsidian: into a page. Same expression syntax, very different operational shape.

Core (this page) REST (port 9179)
Reads from Markdown files on disk. The Obsidian app's Local REST API plugin.
Setup Node server + vault path. Obsidian doesn't need to be installed or running. Install the Local REST API plugin in Obsidian, configure an API key, and point the Node server at it.
Search Substring grep over title, tags, body, path. Obsidian's own search engine — relevance ranking, full-text, fuzzy matching, the metadata cache (backlinks, tag aliases, link graph).
Best for When you don't want the search server itself to depend on Obsidian running — the markdown directory might not even be an Obsidian vault. Simple substring matches are enough. When Obsidian is already running on the same machine and you want richer search (relevance ranking, full-text, fuzzy) or to leverage Obsidian's own indexes (backlinks, link graph, plugin-aware content like Dataview).
Click-through Same in both modes: results carry obsidian://open?vault=…&file=… URIs, so clicking opens the note in Obsidian if you have it installed locally. Override linkTemplate in the protocol config to emit different URLs (e.g., a public-web preview).

The expression syntax is identical — :obsidian:core:foo: here, :obsidian:rest:foo: over there. Switching modes is changing one segment, not rewriting the page.

Where the vault lives

Both example sites (this one and obsidian-rest/) share the vault at examples/sites/obsidian-shared/AlapDocs/. The directory name doubles as the Obsidian vault name — opening that folder in the Obsidian app gives you a vault called "AlapDocs" without renaming, which matches the obsidian://open?vault=AlapDocs URIs the server emits.

The alap-docs portion is regenerated from alap/docs/ via pnpm -w run vault:docs; the topical demo subdirectories (birds/, cooking/, films/, novels/, bach/) come from pnpm -w run vault:demo, which converts the source flavours under examples/sites/obsidian-shared/sources/. Edits made directly in the generated vault won't survive the next regen.

Clicking a result

Each link's URL is an obsidian://open?vault=…&file=… URI. If you have Obsidian installed with the vault configured, your browser will hand the URL to it and the note will open directly.
