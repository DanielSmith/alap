# Conversion sources — example inputs for `vault_convert`

The converter at `scripts/vault_convert.py` (also `pnpm run vault:convert`)
takes arbitrary markdown and produces an Obsidian vault. Each
subdirectory here is a small, self-contained source you can run the
converter against to see what the tool does.

Body content in each source (except `untrusted-demo/`) is adapted
from Wikipedia under [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/);
the SSG-specific syntax wrapping it is mine. Every page carries a
source link at the bottom.

Photos throughout the source flavours come from Wikimedia Commons,
each carrying an inline credit line beneath the image (photographer,
licence, and source). A consolidated list lives in each flavour's
README. Licences vary — most are CC BY-SA 4.0, some are CC BY 4.0,
CC BY-SA 3.0, or public domain.

| Source | Topic | Demonstrates |
|---|---|---|
| [`hugo/`](hugo/) | Birds (a personal birding journal) | Hugo shortcodes — `{{< note >}}`, `{{< figure >}}`, `{{% pageinfo %}}`, `{{% heading %}}`, `{{< toc >}}` |
| [`jekyll/`](jekyll/) | Italian cooking | Jekyll/Liquid — `{% highlight %}`, `{% raw %}`, `{% comment %}`, `{% link %}`, `{{ page.x }}` |
| [`mkdocs/`](mkdocs/) | Hitchcock films | MkDocs admonitions — `!!! note`, `!!! warning "title"`, `??? note`, `???+ note`, `[TOC]` |
| [`docusaurus/`](docusaurus/) | 19th-century English novels | Docusaurus — `:::note`, `:::caution[title]`, MDX imports, `<Tabs>`, heading anchors |
| [`vanilla/`](vanilla/) | Bach (a listening journal) | Plain markdown — no flags, no plugins, baseline behaviour |
| [`untrusted-demo/`](untrusted-demo/) | (converter behaviour, not a topic) | Body HTML, frontmatter HTML, and active-content stripping — what gets removed strict-by-default |

Each source has a README with the exact `pnpm run vault:convert`
command, the flags that matter for that flavour, and a note on what
to look for in the output.

These are *inputs* — running a conversion against any of them writes
its output to a destination of your choice. None of the sources here
ship as vaults. The `AlapDocs/` sibling directory holds the combined
demo vault used by the two example sites; the alap-docs portion is
regenerated by `pnpm -w run vault:docs`, and the topical portion
(`birds/`, `cooking/`, etc.) is regenerated from these very sources
by `pnpm -w run vault:demo`. The two regen scripts touch disjoint
subdirectories of the vault and don't overwrite each other.
